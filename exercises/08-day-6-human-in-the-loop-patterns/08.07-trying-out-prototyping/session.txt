‚óŹ Getting an Ably API key for presence

  1. Create an account

  - Go to ably.com and sign up (free tier is fine ‚ÄĒ 6M messages/month, 200 concurrent
   connections)

  2. Create an app

  - After signing in, you'll land on the dashboard
  - Click "Create New App"
  - Name it something like cadence-dev
  - Select the Free plan

  3. Get the API key

  - Once the app is created, you'll be taken to the app's page
  - Go to the "API Keys" tab (left sidebar)
  - There will be a default key already created (called something like "Root key")

  4. Set the right capabilities

  The default root key has all capabilities enabled. You only need two:

  - Click the key to edit it (or create a new one with "Add new API key" for tighter
  scoping)
  - Under Capabilities, ensure only these are checked:
    - subscribe ‚ÄĒ client receives presence events
    - publish ‚ÄĒ not strictly needed for presence-only, but Ably bundles it
    - presence ‚ÄĒ enter/leave the presence set
    - presence-subscribe ‚ÄĒ receive others' presence events (this is the critical one
  that's easy to miss, it's separate from subscribe)
  - You can leave the resource/channel filter as * (all channels), or restrict to
  lesson:* if you want to be precise

  TL;DR the 4 capabilities needed: subscribe, publish, presence, presence-subscribe

  If you just use the default root key, all of these are already enabled. Creating a
  scoped key is optional but better practice.

  5. Copy the key

  - The key looks like: xVLyHw.xxxxxx:xxxxxxxxxxxx
  - Copy it

  6. Add to your .env

  ABLY_API_KEY=xVLyHw.xxxxxx:xxxxxxxxxxxx

  Then restart the dev server and hit /dev/presence.
