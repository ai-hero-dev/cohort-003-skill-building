# Scoping Document: Write Great PRDs with This Skill

## Scope
Learners use the write-a-prd skill to transform a rough feature idea (instructor analytics dashboard) into a complete, well-scoped PRD by letting Claude interrogate unclear thinking — discovering why clarity in requirements matters for working with AI coding agents.

## Prerequisites
- Understands what skills are and how to invoke them in Claude Code (from 03.01-03.02 earlier in course)
- Knows what a "massive task" is and why it needs planning (from 04.01)
- Has used Claude Code to work with an agent on a real task (from Day 1-2)

## In Scope
- Invoking the write-a-prd skill for the first time on a real feature idea
- Understanding what a complete PRD contains: scope, success criteria, priority, phasing boundaries, edge cases
- Discovering how constraint-driven questioning (the skill's interrogation) surfaces hidden assumptions
- Producing a finished, high-quality PRD ready to hand to 04.03 (multi-phase planning)
- The meta skill: clarity in requirements emerges from disciplined thinking, not upfront perfection

## Out of Scope
- Learning how to break a PRD into multiple phases (that's 04.03's job)
- Executing code or building the feature (that comes in 04.04+)
- Deep business logic for analytics metrics (the skill keeps this lightweight)
- Teaching how skills work mechanically (they already know this)
- Comparing different PRD templates or methodologies (this is not a "PRD frameworks" lesson)

## Teaching Sequence
1. **State the rough feature idea** — "We want to build an instructor analytics dashboard tracking: Revenue trends, Enrollment, Completion rates, Quiz pass rates, Drop-off points" — this is intentionally under-specified
2. **Invoke the skill** — show learners exactly how to call the write-a-prd skill with this idea
3. **Let the skill interrogate** — Claude will ask: Who uses this? Which metrics first? What's success? What's out of scope? Let this unfold naturally
4. **Watch clarity emerge** — learners discover that answering those questions transforms vague ambition into concrete, scoped work
5. **Review the finished PRD** — show what a "done" PRD looks like and why 04.03 can now take it and confidently break it into phases
6. **Reflect on the meta skill** — brief callout: "This clarity work upfront saves massive rework later when you're coding"

---

**Lesson Format Note**: This is a hands-on, discovery-based exercise. Teacher demonstrates invoking the skill on the instructor analytics idea, learners follow along. Expected duration: 20–25 minutes. Learner exits with a complete PRD artifact.
