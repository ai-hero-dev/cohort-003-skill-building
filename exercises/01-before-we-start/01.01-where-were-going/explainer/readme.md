However, as I was teaching this course, I realized I'm not actually making a Claude Code course here, I'm making an **AI coding course**.

### Real Engineering Skills

The most important thing you're going to get out of this course is real engineering skills.

If you already have real engineering skills, that's fantastic. What I found is that I needed to adapt my engineering skills to work with these new tools. While the tools have changed, the fundamental approach really hasn't.

Think of this course as a **conversion course** - taking your real engineering skills and using these new tools with them.

But if you haven't had the grounding in what real engineering looks like, this course is going to be great for you too.

## Building Trustworthy AI Agents

The final form of this is going to be an **AI agent that we can trust to run AFK** - in other words, while we're away from the keyboard doing other things.

One of the things people say about AI is that they have to sit there and watch it, which feels less productive. That's because they haven't set up their codebase and environment to run AI properly and autonomously.

Getting there and guaranteeing quality at the end of each AFK run is difficult. That's where the real engineering skills come in - you need to make your codebase something that AI likes to work in and produces quality code in. That's often more about the code than the AI.
