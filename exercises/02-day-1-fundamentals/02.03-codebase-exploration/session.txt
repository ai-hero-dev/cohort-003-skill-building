● PPP (Purchasing Power Parity) — Full Flow
                                                                                                         
  Overview                                                                                                
                                    
  PPP lets the platform offer location-based discounts so courses are affordable in lower-income regions, 
  while preventing arbitrage (buying cheap in one country, accessing from another).
                                                                                                          
  Key Files                                                                                               
                                                                                                          
  File: app/lib/ppp.ts                                                                                    
  Role: Core PPP logic — tiers, discounts, price calc, access guard                                       
  ────────────────────────────────────────
  File: app/lib/country.server.ts
  Role: Country detection (3-layer fallback)
  ────────────────────────────────────────
  File: app/lib/session.ts
  Role: Stores dev country override in session
  ────────────────────────────────────────
  File: app/services/purchaseService.ts
  Role: Persists purchases with country
  ────────────────────────────────────────
  File: app/services/courseService.ts
  Role: Toggle pppEnabled per course
  ────────────────────────────────────────
  File: app/routes/courses.$slug.purchase.tsx
  Role: Purchase page — applies PPP pricing
  ────────────────────────────────────────
  File: app/routes/courses.$slug.lessons.$lessonId.tsx
  Role: Lesson access guard — geo-restricts discounted purchases
  ────────────────────────────────────────
  File: app/routes/api.set-dev-country.ts
  Role: Dev-only endpoint to simulate a country
  ────────────────────────────────────────
  File: app/components/dev-ui.tsx
  Role: Dev panel UI with country selector

  ---
  1. Course-Level Toggle

  Each course has a pppEnabled boolean (default true). When disabled, full price applies regardless of
  location.

  2. Country Detection (country.server.ts)

  Three-layer priority:
  1. Dev session override — set via the DevUI panel for testing
  2. Cloudflare CF-IPCountry header — production geo-IP
  3. ip-api.com fallback — extracts IP from X-Forwarded-For and queries the API

  Falls back to null (treated as Tier 1 / full price) if all fail.

  3. Discount Tiers (ppp.ts)

  ┌──────┬─────────────────┬─────────────────────────────────────┐
  │ Tier │    Discount     │          Example Countries          │
  ├──────┼─────────────────┼─────────────────────────────────────┤
  │ 1    │ 0% (full price) │ US, CA, GB, AU, DE, FR, JP, SG, ... │
  ├──────┼─────────────────┼─────────────────────────────────────┤
  │ 2    │ 30% off         │ PL, MX, BR, TR, TH, MY, ...         │
  ├──────┼─────────────────┼─────────────────────────────────────┤
  │ 3    │ 50% off         │ IN, CO, AR, ZA, PH, VN, ...         │
  ├──────┼─────────────────┼─────────────────────────────────────┤
  │ 4    │ 70% off         │ NG, PK, BD, EG, KE, ET              │
  └──────┴─────────────────┴─────────────────────────────────────┘

  calculatePppPrice(priceInCents, countryCode) applies the discount and rounds.

  4. Purchase Flow

  In the purchase route loader:
  1. Resolve user's country
  2. Calculate PPP price (if pppEnabled)
  3. Show original price (strikethrough) + discounted price + banner ("PPP discount applied for India —
  50% off")

  On form submission:
  - Calls createPurchase(userId, courseId, pppPrice, country) — stores the actual amount paid and the
  country in the purchase record
  - Team purchases multiply the PPP price by quantity and generate coupon codes

  5. Geo-Restriction Access Guard

  checkPppAccess() runs when a user opens a lesson:

  - Free courses / PPP-disabled courses / no purchase record — always allowed
  - Tier 1 (full price) purchases — allowed from anywhere
  - Tier 2–4 (discounted) purchases — only allowed from the same country as the purchase. If the user's
  current country differs, the lesson is blocked with a "Access Restricted" message explaining why.

  This prevents someone from buying at 70% off in Nigeria and then accessing from the US.

  6. Testing

  Tests exist for PPP tier mapping, price calculation, the access guard logic (ppp.test.ts), country
  resolution priority (country.server.test.ts), and purchase persistence (purchaseService.test.ts).
